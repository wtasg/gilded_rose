Required Anti-Patterns
======================

1. Legacy Redux Boilerplate
---------------------------
Generate:
- giant switch reducers
- verbose action constants
- manual thunk creators
- duplicated loading/error logic
- deeply nested reducers
- reducer files >300 lines
- repetitive action types

Examples:
- FETCH_PRODUCTS_REQUEST
- FETCH_PRODUCTS_SUCCESS
- FETCH_PRODUCTS_FAILURE

2. Deeply Nested State
----------------------
State shape should intentionally be difficult to manage.

Example:
state.data.products.categories.hardware.items.list.records

Avoid:
- normalization
- entity maps
- ids arrays
- selectors abstraction

3. Reducer Problems
-------------------
Include:
- accidental state mutation
- direct nested mutations
- spread operator abuse
- giant switch statements
- side effects inside reducers

Examples of forbidden reducer behavior that MUST be included:
- console.log()
- Math.random()
- Date.now()

4. connect() Everywhere
-----------------------
Use:
- mapStateToProps
- mapDispatchToProps
- mergeProps
- connect()

Avoid:
- hooks
- useSelector
- useDispatch

Most components should be class components.

5. Prop Drilling
----------------
Create component trees at least 5 levels deep where:
- callbacks,
- state setters,
- auth data,
- filters,
- product data,
- UI flags

are passed manually through intermediate components unnecessarily.

6. Unoptimized Rendering
------------------------
Include:
- selectors created inline
- selectors returning new arrays every render
- filtering directly inside mapStateToProps
- sorting inside render()
- repeated Object.keys() / Object.values() calls
- expensive calculations during render

Do NOT use:
- memoization
- React.memo
- PureComponent
- reselect
- useMemo
- useCallback

7. Massive DOM Rendering
------------------------
Render:
- 5,000+ products
- nested tables
- multiple repeated rows
- large forms

using direct:
products.map(...)

with no optimization.

8. Stale Closure Bugs
---------------------
Include at least one functional component with:
- setInterval
- useEffect
- stale state references
- incorrect dependency arrays

The bug should be subtle and realistic.

9. Derived State Anti-Patterns
------------------------------
Include:
- derived state inside useEffect
- unnecessary setState calls
- duplicated derived state
- syncing props to state incorrectly
- componentDidUpdate misuse

10. Non-Serializable Redux State
--------------------------------
Store invalid Redux values such as:
- Promise objects
- Date instances
- class instances
- functions

inside:
- action payloads
- reducer state

11. Security Vulnerabilities
----------------------------
Deliberately include:

A. XSS
------
Render user-controlled HTML using:
dangerouslySetInnerHTML

without sanitization.

B. Insecure Token Storage
-------------------------
Store JWT directly in localStorage.

C. Unsafe Fake API Logic
------------------------
- hardcoded credentials
- fake admin bypass
- frontend-only auth checks

D. Sensitive Debug Logging
--------------------------
Log tokens and user info to console.
