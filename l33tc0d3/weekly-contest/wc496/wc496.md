# Weekly Contest 496

## 1. [mirror frequency distance(https://leetcode.com/problems/mirror-frequency-distance/)

+ [3889-mirror-frequency-distance-1.ts](/l33tc0d3/weekly-contest/wc496/3889-mirror-frequency-distance-1.ts)
+ [3889-best-solution.ts](/l33tc0d3/weekly-contest/wc496/3889-best-solution.ts)

Enhancements

1. Iterating over `freq.keys()` is better than iterating over `str` input; 330ms->200ms.
2. Using `seen` as a `Set` instead of `{} as Record` is a bit faster still; 200ms->190ms.
3. When using frequency, it is better to just have an array large enough that covers the range (say ascii).

### Don't do: Below is too clever code

```ts
function mirrorFrequency(s: string): number {
    const f = new Uint32Array(128);
    for (let i = 0; i < s.length; i++) f[s.charCodeAt(i)]++;
    let d = 0;
    for (let i = 0x30; i <= 0x34; i++) d += Math.abs(f[i] - f[0x69 - i]);
    for (let i = 0x61; i <= 0x6d; i++) d += Math.abs(f[i] - f[0xdb - i]);
    return d;
}
```

### 2. [3890. Integers With Multiple Sum of Two Cubes](https://leetcode.com/problems/integers-with-multiple-sum-of-two-cubes/description/)

+ [3890-sum-of-two-cubes.ts](/l33tc0d3/weekly-contest/wc496/3890-sum-of-two-cubes.ts)
+ [3890-optimized.ts](/l33tc0d3/weekly-contest/wc496/3890-optimized.ts)

This is about Ramanujan Taxi numbers i.e. multiple a3+b3 for one number.

Optimizations

1. Store only the count instad of factor pairs.
2. Use `Math.cbrt(n)` to get new limit for current number. No need to generate cubes for numbers we won't be reaching.
